# FreeMail v0.1.0-private-beta

FreeMail v0.1.0-private-beta is the first private-beta candidate for the AGPL-licensed FreeMail stack. It is intended for VPN-only validation on `freemail.kuzuryu.ai` before any public release or hosted external-user availability.

## Included

- Email/password login, MFA setup, invitation-based onboarding, and administrator bootstrap controls.
- Backend API surfaces for accounts, domains, mailbox metadata, web sessions, mobile session bootstrap, push-device registration, admin operations, deployment metadata, and readiness checks.
- Web mail UI shell with inbox, compose, thread, search, settings, and admin workflows.
- Mobile client foundation for iOS and Android with environment-driven API targeting, authentication, mailbox views, composer, push registration, and release-evidence gates.
- Docker Compose deployment for the API, web client, and Stalwart-backed mail-core spike.
- CI coverage, repository secret scan, AGPL-compatible direct dependency license-policy scan, static QA, mobile QA, backup/restore tooling, release gate, private-beta gate, and mobile release gate.

## Verification

Release evidence for this candidate must include passing GitHub Actions CI for the exact commit, Codecov upload completion, repository secret/signing-material scan completion, direct dependency license-policy scan completion, `docker compose config --quiet`, metadata backup evidence, mail-store backup evidence, VPN-only runtime health, deployment-boundary evidence, metadata readiness evidence, mail-core readiness evidence, private-beta DNS/mail-flow evidence for every controlled domain, and this release-notes checksum in the release-gate JSON output.

## Known Limitations

- The deployment boundary remains VPN-only. Do not expose this candidate directly to the public internet.
- SQLite is the only supported API metadata backend. PostgreSQL requires a database adapter, migrations, backup/restore coverage, and release-gate evidence before it can be claimed as supported.
- Stalwart mail-core integration is still a private-beta spike until controlled-domain DNS, mail-flow, queue, deliverability, abuse, backup, and owner-acceptance evidence are recorded.
- Real signed native builds, app-store submission evidence, and store approval are not complete release evidence until `scripts/mobile_release_gate.py` passes with current credential-free evidence.
