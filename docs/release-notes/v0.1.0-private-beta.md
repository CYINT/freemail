# FreeMail v0.1.0-private-beta

FreeMail v0.1.0-private-beta is the first private-beta candidate for the AGPL-licensed FreeMail stack. It is intended for VPN-only validation on `freemail.kuzuryu.ai` before any public release or hosted external-user availability.

## Included

- Email/password login, MFA setup, invitation-based onboarding, and administrator bootstrap controls.
- Backend API surfaces for accounts, domains, mailbox metadata and quota metadata, web sessions, mobile session bootstrap, push-device registration, admin operations, deployment metadata, and readiness checks.
- Web mail UI shell with inbox, compose, draft saving/editing, reply/forward, read/unread and star state, bulk message actions, archive/spam/delete, search, contacts, attachment download, Sent Items persistence status, and admin workflows including mailbox quota metadata.
- Mobile client foundation for iOS and Android with environment-driven API targeting, authentication, mailbox views, composer, draft saving/editing, read/unread and star state, bulk message actions, Sent Items persistence status, push registration, and release-evidence gates.
- Docker Compose deployment for the API, web client, and Stalwart-backed mail-core spike.
- CI coverage, required CI step provenance, repository secret scan, AGPL-compatible direct dependency license-policy scan, static QA, mobile QA, backup/restore tooling, release gate, private-beta gate, Stalwart apply evidence collector, and mobile release gate.

## Verification

Release evidence for this candidate must include passing GitHub Actions CI for the exact commit, Codecov upload completion, repository secret/signing-material scan completion, direct dependency license-policy scan completion, `docker compose config --quiet`, metadata backup evidence, mail-store backup evidence, restore-drill evidence, mobile signed-build/store-submission evidence, private-beta gate output for each controlled domain, VPN-only runtime health, deployment-boundary evidence, metadata readiness evidence, mail-core readiness evidence, private-beta DNS/mail-flow/queue/mail-core-apply/restore-drill evidence for every controlled domain, and this release-notes checksum in the release-gate JSON output.

## Known Limitations

- The deployment boundary remains VPN-only. Do not expose this candidate directly to the public internet.
- SQLite is the only supported API metadata backend. PostgreSQL requires a database adapter, migrations, backup/restore coverage, and release-gate evidence before it can be claimed as supported.
- Stalwart mail-core integration remains private-beta scoped until decision-owner acceptance is recorded and every release candidate refreshes controlled-domain DNS, mail-flow, queue, mail-core apply, deliverability, abuse, backup, and restore evidence.
- Real signed native builds, app-store submission evidence, and store approval are not complete release evidence until `scripts/mobile_release_gate.py` passes with current credential-free evidence.
